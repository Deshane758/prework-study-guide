HTML

  • Some HTML elements provide important information about a website to the browser, search engine and other developers
  • Elements such as and elements are nested in the <head> element</head>
  • Elements nested in other elements are called "child elements". The element that holds all the child elements is known as the "parent element"
  • The element is an example of a semantic HTML element - The name of the element describes its contents and the role it plays on a webpage.
  • Semantic HTML elements ensure your webpage is accessible to the masses including those who use assistive technologies like screen readers.
  • HTML content is contained between an opening and closing tag represented by '<>' & ''
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user

CSS-Section

  • Css is used to add graphics to the user interface and to make the content on the page more appealing.
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • A selector defines the element or attributes to which the rules, or declarations, will apply
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • Inline styles apply to a special HTML tag, using a style attribute with a CSS rule to style a particular page element only
  • Inline CSS and internal style sheets are totally acceptable methods under certain conditions, but the most common way to style a webpage is by using an external CSS style sheet
  • Separation of concerns is a widely used term in web development and refers to separating code into different files based on use.

Git

  • Git is used for version control
  • Git enables developers to contribute/work on the same project remotely and simultaneously without causing confusion
  • Git makes it easy for other developers to read and understand what another developer has done to a code by allowing the developer to add comments to his/her code.
  • Git allows you to work on as many feature branches as you may want locally on your machine before you push to the main branch
  • You must raise a PR which has to be reviewed and approved before you can push your code to the main branch
  • Feature branches are used to add to the code source, and you're able to have as many features are you want in a develop branch before deploying code
  • Git status checks what branch you're in
  • git checkout -b branch-name, creates a branch and swithces to it
